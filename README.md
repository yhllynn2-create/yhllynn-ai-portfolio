# 销售 AI 提效

> 用 AI 把销售日常最耗时的"非销售动作"自动化，让销售把时间花在客户和商机上。

本仓库是一个实战项目集，主题是「销售团队的 AI 提效」。目前包含两个已落地的子项目（企微每周话题话术引擎、每日拓客引擎），更多场景在规划中。每个子项目都是一套可落地、可复用的 AI 工作流（以 WorkBuddy Skill 形式封装），来源于真实销售场景，经过实际周会汇报与团队使用验证。

> **机密与脱敏**：本仓库默认按"可公开"标准脱敏——**不收录任何真实客户名单、真实公司数据、内部系统名**。各子项目均以"制造业 B2B 销售"为示例场景演示方法论（框架通用，可套用任意行业的企微运营），真实客户名单由各团队本地保管、绝不入库（见各子项目说明与 `.gitignore`）。

## 为什么做这个仓库

- 销售最头疼的不是"不会卖"，而是**每周要花大量时间想发什么、写什么、整理什么**。
- 把这些"低杠杆但高频"的动作交给 AI 引擎，产出标准化素材，销售只做"复制粘贴 + 人性化跟进"。
- 沉淀为开源结构，方便复用、迭代，也便于复盘与团队同步。

## 子项目清单

| 子项目 | 解决的问题 | 形态 |
|--------|-----------|------|
| [销售企微运营](./销售企微运营/) | 每周不知道发什么给客户 / 发了像群发 | WorkBuddy Skill + 每周自动化 |
| [每日拓客](./每日拓客/) | 每天去哪找"值得跟"的新客户 / 如何不撞上在库存量客户 | WorkBuddy Skill + 强排除校验脚本 |

> 规划中（尚未落地）：客户跟进提醒、商机线索自动打分、周报/日报自动生成、竞品动态监控……

## 适用人群

- B2B 销售 / 销售运营 / 销售管理者
- 想用 AI 提效但团队"每人部署难"的情境（本仓库采用**中心化生产 + 全员分发**架构，1 人跑引擎、全员复制粘贴）

## 如何参与

1. Fork / Clone 本仓库
2. 进入对应子项目目录阅读 `README.md` 与 `SKILL.md`
3. 按子项目说明部署到 WorkBuddy 或直接复用方法论
4. 优化后提 PR

> 详细内容规范与提交流程见 [CONTRIBUTING.md](./CONTRIBUTING.md)。
> 建议 GitHub 仓库名用英文（如 `sales-ai-boost`），本仓库目录保留中文便于阅读。
