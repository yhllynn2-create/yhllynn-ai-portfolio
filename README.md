# 销售 AI 提效

> 用 AI 把销售日常最耗时的"非销售动作"自动化，让销售把时间花在客户和商机上。

本仓库是一个**持续生长的实战项目集**，主题是「销售团队的 AI 提效」。每一个子项目都是一套可落地、可复用的 AI 工作流（多数以 WorkBuddy Skill 形式封装），来源于真实销售场景，经过实际周会汇报与团队使用验证。

## 为什么做这个仓库

- 销售最头疼的不是"不会卖"，而是**每周要花大量时间想发什么、写什么、整理什么**。
- 把这些"低杠杆但高频"的动作交给 AI 引擎，产出标准化素材，销售只做"复制粘贴 + 人性化跟进"。
- 沉淀为开源结构，方便复用、迭代，也方便在转正答辩 / 团队分享时展示方法论。

## 项目结构

```
销售AI提效/
├── README.md                # 本文件：项目总览
└── 销售企微运营/             # 子项目 ①：企微每周话题话术引擎（WorkBuddy Skill）
    ├── README.md            # 子项目说明（用法 / 架构 / 示例）
    ├── SKILL.md             # Skill 本体（5 步工作流 + 触发词）
    ├── references/          # 知识底座：话题话术库 / 输出模板 / 话题历史
    └── 示例/                # 实样产出：话题包 + 周会方案 + 汇报 PPT
```

|--------|-----------|------|
| [销售企微运营](./销售企微运营/) | 每周不知道发什么给客户 / 发了像群发 | WorkBuddy Skill + 每周自动化 |

## 适用人群

- B2B 销售 / 销售运营 / 销售管理者
- 想用 AI 提效但团队"每人部署难"的情境（本仓库采用**中心化生产 + 全员分发**架构，1 人跑引擎、全员复制粘贴）

## 如何参与

1. Fork / Clone 本仓库
2. 进入对应子项目目录阅读 `README.md` 与 `SKILL.md`
3. 按子项目说明部署到 WorkBuddy 或直接复用方法论
4. 优化后提 PR
