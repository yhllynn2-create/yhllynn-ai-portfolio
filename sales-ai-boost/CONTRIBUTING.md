# 如何参与贡献

欢迎 Fork / 提 PR 来完善这套「销售 AI 提效」方法论。

## 仓库结构约定
- 每个子项目放在独立目录，含 `README.md` + `SKILL.md`（如为 WorkBuddy Skill）+ `references/` + `示例/`。
- 文件名统一用**短横线英文**（如 `企微话题引擎_落地方案.md` 这类中文名仅示例保留，新增文件请避免空格与过长名）。

## 贡献流程
1. Fork 本仓库到你的账号。
2. 新建分支：`git checkout -b feat/你的改动`（如 `feat/wecom-more-industry`）。
3. 在对应子项目里修改：
   - 改话术/矩阵 → 编辑 `references/话题话术库.md`
   - 改工作流 → 编辑 `SKILL.md`
   - 加示例 → 放进 `示例/`，并在子项目 `README.md` 的"文件结构"里登记
4. 本地自测：用 WorkBuddy 加载 `SKILL.md` 跑一次，确认话题包可生成、无 P0–P4 等已废弃概念残留。
5. 提交并 Push，开 PR 描述清楚"改了什么 / 为什么 / 适用场景"。
6. 维护者 Review 后合并。

## 内容规范
- 不提交任何内部专属文件名、真实客户名称、内部系统名等敏感信息（本项目默认按"可公开"标准脱敏）。
- 话术卡字段不使用 emoji 做结构标记；emoji 仅可出现在发给客户的话术正文（1–2 个）。
- 客户定位统一用"行业 / 岗位"，不使用 P0–P4 等内部分层标签。

## 新增子项目
如有新的销售提效场景（客户跟进提醒、周报生成、竞品监控等），请新建同级目录，参照 `销售企微运营/` 的结构组织，并在根 `README.md` 的"子项目清单"中登记。
