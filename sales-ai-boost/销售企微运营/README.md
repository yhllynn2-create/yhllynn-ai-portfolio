# 销售企微运营 · 企微每周话题话术引擎

> 一条命令，产出《本周话题包》：5 个当周最新鲜、最相关的话题，每个都配好「可直接复制发企微」的话术卡片。

本子项目是一个 **WorkBuddy Skill**（`lenovo-wecom-weekly-topics`），解决 B2B 销售一个高频痛点：**每周发企业微信给客户时，不知道发什么、写出来像群发通稿**。

## 落地架构：中心化生产 + 全员分发

```
┌─────────────┐   产出话题包    ┌──────────────┐   复制粘贴   ┌──────────┐
│  1 人跑引擎  │ ─────────────▶ │ 企微群/腾讯文档 │ ──────────▶ │ 全体销售 │
│ (运营/负责人) │  每周一 09:00  │  (统一分发)   │  零部署零学习 │ (只发不写)│
└─────────────┘                └──────────────┘             └──────────┘
```

- 只需 **1 人** 每周一触发一次，全体销售**零部署、零学习**，直接复制粘贴。
- 内容统一、可控、过红线自检，避免"每人各写各的、质量参差"。

## 核心能力

### 1. 话题 8 类矩阵（保证内容不重样、有梯度）

| # | 类别 | 发出目的 | 联想衔接点 | 承接商机 |
|---|------|---------|-----------|---------|
| 1 | 联想好消息 | 证明实力、引入合作 | 新品/技术/全栈能力 | 产品切入 |
| 2 | 行业趋势/政策 | 问需求、显专业 | 数字化/AI 方案 | 数字化·算力 |
| 3 | 客户企业专属喜讯 | 恭喜、拉近关系（须真实） | 扩产要设备跟上 | 扩产采购 |
| 4 | 政策补贴/以旧换新 | 「有笔钱你能省」 | 批量换新/TruScale | 批量换新 |
| 5 | 实用干货 tips | 立专家人设 | 可管理/安全/降本 | 长期信任 |
| 6 | 节日/节气/关怀 | 纯暖场、无压力 | 不推产品 | 维系关系 |
| 7 | 联想活动/权益 | 直接转化钩子 | 活动/优惠/会员 | 报名·询价 |
| 8 | AI/算力热点借势 | 蹭趋势、聊痛点 | AI PC/工作站/算力 | 工作站·算力 |

### 2. 五段式话术公式（五要素）

```
开场关怀 ＋ 事件钩子 ＋ 联想衔接 ＋ 价值锚点 ＋ 轻 CTA
```

- 开场关怀：称呼 + 一句人情味问候
- 事件钩子：本周话题，带时效感（「刚」「最新」）
- 联想衔接：自然过渡到能力，不硬广
- 价值锚点：只推一个（TCO 更省 / 数据不出厂）
- 轻 CTA：低压力下一步（「发份资料参考?」）

### 3. 四条红线（每次必过）

- 不编造客户事件：专属喜讯须真实可核实来源
- 不承诺具体价格：只说「有批量方案/专属价」
- 单信息原则：一条话术只推一个核心信息
- 来源可溯：素材留链接（内部留档）；关怀类只暖场不硬推

## 工作流（5 步）

1. **定基准**：确定本周周次、临近节日/节气、指定客户行业
2. **联网抓**：WebSearch 抓近 7–14 天新鲜素材，8 类每类至少 1 次
3. **筛 5 话题**：挑最新鲜+最相关，读 `话题历史.md` 错开上周类别
4. **生成卡**：按五段式话术公式 + 话术库语气，产出 5 张话题卡
5. **红线自检 + 输出**：过四条红线 → 生成《本周话题包_YYYY-MM-DD.md》→ 追加类别到历史 → 交付

## 如何部署为 WorkBuddy Skill

```bash
# 把本目录整体复制到 WorkBuddy 用户级 skills 目录即可被识别
cp -r 销售企微运营 ~/.workbuddy/skills/lenovo-wecom-weekly-topics
```

部署后在对话中说「生成本周企微话题包」即可触发；也可配置每周一 09:00 自动化定时产出。
