# 每日拓客引擎 · 新客户筛选与调研流水线

> 一条命令，产出一批"大而近期有信号"、且**绝不撞上你在库存量客户**的新客户，附标准化《公司调研报告》+ 汇总表。

本子项目是一个 **WorkBuddy Skill**（`sales-daily-prospecting`），解决 B2B 销售两个高频痛点：**① 每天去哪找"值得跟"的新客户；② 如何保证不把已经在跟的存量客户当成新客重复报**。

> 示例场景：某 B2B 厂商（示例：某头部硬件厂商）的深圳制造业销售，每天补 10 家"大而近期有信号"的新制造客户，且必须排除已在库的两千多家存量客户。本 Skill 把这套打法抽象成任何人可套用的流程——**方法开源，你的客户名单本地保管、绝不入库**。

## 落地架构：中心化生产 + 全员分发

```
┌─────────────┐  产出线索包   ┌──────────────┐  复制跟进   ┌──────────┐
│  1 人跑引擎  │ ───────────▶ │ 汇总表/云文档  │ ──────────▶ │ 全体销售 │
│ (销售运营)   │  每个工作日   │  (统一分发)   │  零部署零学习 │ (只跟不写)│
└─────────────┘              └──────────────┘             └──────────┘
```

- 只需 **1 人** 按节奏触发，全体销售**零部署、零学习**，直接跟进。
- 强排除红线统一执行，避免"每人各筛各的、重复撞库"。

## 核心能力

### 1. 信号驱动候选采集
围绕"有资金 / 有扩编 = 有采购"逻辑，从招聘、融资、IPO、招投标、扩产新闻捞候选（详见 `references/信号源清单.md`）。

### 2. 强排除校验（本 Skill 的护城河）
对候选与你团队的**在库/存量客户名单**做四重比对，零命中才过：
- 归一化（去城市前缀/法律后缀/括号）
- 双向子串匹配
- `difflib` 模糊兜底（≥0.85）
- 别名/母子公司穿透（如"示例控股↔示例科技"）

算法与可执行脚本见 `references/强排除校验方法.md` 与 `scripts/strong_exclusion.py`。

### 3. 7 节标准化调研报告
公司概况 / 工商与股权 / 财务表现 / 招聘信号 / 融资上市股价信号 / IT 设备与算力需求 / 切入建议（针对你公司产品线）。模板见 `references/调研报告模板.md`。

## 工作流（6 步）

1. **定基准**：目标区域/行业/规模门槛/时间窗口/在库名单路径
2. **采候选**：从信号源捞"规模达标+近期有信号"的公司
3. **强排除**：对在库名单四重比对，零命中才过（硬红线）
4. **深核验**：工商注册地+规模+信号多源交叉验证，确认在区域、真实、不重复
5. **写报告**：按 7 节模板产出每家报告（干净工商全称）
6. **交付**：markdown 源 + docx 备份 + 汇总表追加 +（可选）云文档分发

## 5 分钟跑通（quickstart）

不想读文档也能立刻跑通？见 [`示例/quickstart.md`](./示例/quickstart.md) —— 用仓库自带的虚构演示数据，从「候选名单」一路跑到「docx 报告 + 汇总表 xlsx」，零真实数据依赖。

## 如何部署为 WorkBuddy Skill

```bash
# 把本目录整体复制到 WorkBuddy 用户级 skills 目录即可被识别
cp -r 每日拓客 ~/.workbuddy/skills/sales-daily-prospecting
```

部署后在对话中说「做今日拓客，找 10 家新客户」即可触发；也可配置每个工作日 09:00 自动化。

## ⚠️ 机密与脱敏（非常重要）

- **你的在库/存量客户名单（含数千家公司名）属于商业机密，本仓库一律不收录**。脚本默认读取 `示例/incumbents_sample.json`（明显虚构的演示数据）。
- 真实名单请放在仓库**之外**的本地路径，运行 `strong_exclusion.py --list /你的/本地/名单.json` 时传入。
- 根目录与本目录的 `.gitignore`（或根 `.gitignore`）已加入排除规则，即使 `git add .` 也不会把真实名单误提交。
- 本 Skill 所有示例公司均为虚构，不含任何真实客户信息。

## 已知可优化点（待迭代）

- 别名表 `ALIAS` 需各团队按自身客户结构本地维护（不入库）。
- 信号采集目前依赖 WebSearch + 企查查/天眼查 MCP；可接入更多数据源（如招投标 API）。
- 汇总表列结构为通用 10 列；如需对齐你部门既有大表，调整 `build_report.py` 的表头即可。
