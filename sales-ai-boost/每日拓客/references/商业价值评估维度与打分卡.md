# 商业价值评估维度与打分卡

> 本打分卡是「客户商业价值研判」能力的核心。它把一次客户调研得到的**定性事实**，转成**可比较、可排序、可决策**的量化层级（S / A / B / C），让"这家值不值得跟、值多少资源"从感觉变成方法。
>
> 它同时服务于本 Skill 的两种模式：
> - **拓客模式**：对通过强排除的新候选跑打分卡 → 本批线索天然按价值分层，先打高价值的。
> - **研判模式**：对任意已知公司（存量客户 / 重点目标 / 竞争对手客户）跑打分卡 → 输出经营价值结论与资源建议。

---

## 一、10 个评估维度与评分标准

每个维度打 **1–5 分**（5=最强，1=最弱），按权重加权求和。权重合计 100%。

| # | 维度 | 权重 | 5 分（强） | 3 分（中） | 1 分（弱） | 数据来源 |
|---|------|------|-----------|-----------|-----------|----------|
| 1 | 经营稳定性（成立时间） | 8% | ≥10 年且存续稳健 | 5–10 年 | <3 年或频繁更名/迁址 | 工商信息 |
| 2 | 资本化 / 上市 | 10% | 已上市 / 已过会拟上市 | 已完成 B 轮后多轮融资 | 天使期 / 未融资早期 | 工商 / 融资公告 |
| 3 | 人员规模 | 12% | ≥2000 人 | 200–2000 人 | <200 人 | 招聘平台 / 工商年报 |
| 4 | 主营业务与卖方匹配度 | 13% | 强相关、高客单、设备/算力密集 | 部分相关 | 弱相关 / 低客单 | 行业分类 + 产品映射 |
| 5 | 盈利情况 | 13% | 持续盈利且增长 | 盈亏平衡 / 小幅波动 | 持续亏损 | 财报 / 招股书 / 公告 |
| 6 | 工厂 / 产能分布 | 10% | 多基地自有产能（扩产中） | 单基地自有产能 | 纯贸易 / 轻资产无产线 | 工商 / 官网 / 新闻 |
| 7 | 行业地位 | 12% | 龙头 / 细分冠军 / 标准制定者 | 区域前列 / 专精特新 | 长尾跟随者 | 行业报告 / 招投标 |
| 8 | 近期发展动能 | 7% | 扩产 / 中标大单 / 新基地动工 | 平稳运营 | 收缩 / 退出业务 | 招投标 / 新闻 |
| 9 | 未来赛道趋势 | 7% | 朝阳高景气（政策+需求双升） | 成熟稳定 | 下行 / 被替代风险 | 行业研判 / 政策 |
| 10 | 近期招聘 | 8% | 大规模扩招（新人→新设备） | 常规补员 | 裁员 / 招聘冻结 | 招聘平台 |

**权重合计 = 100%**　（8+10+12+13+13+10+12+7+7+8 = 100）

## 二、加权总分与层级阈值

加权总分 = Σ（维度分 × 权重），满分 5.0。

| 层级 | 加权总分 | 含义 | 资源建议 |
|------|----------|------|----------|
| **S 级 · 战略 / 重点攻坚** | ≥ 4.0 | 高价值、强信号，值得高层直触 | 高层直触 / 定制化方案 / 优先资源倾斜 / 专人对接 |
| **A 级 · 优先跟进** | 3.2 – 3.99 | 价值明确，主力跟进对象 | 主力销售跟进 / 标准方案 + 批量折扣 / 季度复盘 |
| **B 级 · 常规培育** | 2.4 – 3.19 | 有潜力但信号偏弱 | 纳入培育池 / 定期触达 / 轻量方案 / 事件触发再升级 |
| **C 级 · 低优先 / 观察** | < 2.4 | 当前价值有限 | 仅留存信息 / 低优先 / 重大信号出现再激活 |

## 三、红旗规则（软约束，不自动判级）

以下情形即使加权分偏高，也**不得直接判 S**，须人工复核：
- 任一维度 = **1 分**（如持续亏损、裁员冻结、赛道下行、纯贸易无产能）；
- 近 12 个月有**重大负面**（行政处罚、核心高管离职、债务违约、核心客户流失）；
- 拓客模式下**强排除命中**（=已在库存量客户）→ 直接剔除，不进入打分。

> 红旗的作用：防止"账面上分数高、实际踩雷"的误判。打分卡给方向，人工给最终裁定。

## 四、打分卡与两种模式的衔接

### 拓客模式（发现新客 + 顺带分层）
```
信号发现候选
   → 强排除（vs 在库名单，零命中）
   → 写 7 节调研报告（事实基底）
   → 跑打分卡 → 标注价值层级
   → 输出"新客 + 价值层级"批次（高价值优先）
```

### 研判模式（评估已知公司）
```
给定公司（存量客户 / 目标 / 竞品客户）
   → 写 7 节调研报告（事实基底）
   → 跑打分卡 → 标注价值层级
   → 输出"经营价值结论 + 资源建议"
```

两种模式**共用同一份 7 节事实基底**与同一张打分卡，区别只在入口（信号 vs 指定公司）和出口（线索包 vs 研判结论）。这正是一个项目能同时干两件事的关键：**采集与报告是共享引擎，打分卡是价值判定的统一标尺**。

## 五、可调参与落地说明

- **权重可调**：不同行业 / 不同产品线，对"匹配度""产能""盈利"的侧重不同。把本表权重改成适合你团队的版本（保持合计 100%），存到本地即可。
- **分数谁来打**：可由分析师按调研事实人工打分；也可在前端接规则（如"已上市=5，B轮=3，天使=1"）半自动预填，人工校准。
- **脚本化**：见 `scripts/value_score.py` —— 输入 `{公司: {维度: 分数}}` 的 JSON，自动算加权总分、判层级、排序、标红旗。
- **脱敏**：本卡所有示例均为虚构；真实客户分数本地保管，不与本仓库同库。
