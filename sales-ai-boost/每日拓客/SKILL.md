# 客户商业价值研判与拓客引擎（sales-daily-prospecting）

## 这个 Skill 是什么
把 B2B 销售两件最耗精力的事，合并成一套**可复用、可审计**的流水线：

1. **拓客（发现新客）**：从海量信号里筛出"大且近期有采购迹象"的公司，严格排除你已经在跟的存量客户，产出能直接进团队客户池的《公司调研报告》+ 汇总表。
2. **研判（评估价值）**：对任意已知公司评估"经营价值几何、值多少资源"，按统一打分卡定级 S/A/B/C，输出结论与资源建议。

> 关键洞察：拓客看"这家新客有没有经营价值"，和研判看"这家客户经营价值如何"，**用的是同一套评估维度**（成立时间、是否上市、人员规模、主营业务、盈利、工厂分布、行业地位、近期发展、未来趋势、近期招聘）。所以本 Skill 让两者共享**同一引擎、同一标尺**——采集与 7 节报告是共享底座，打分卡是价值判定的统一标尺。

> 示例场景：某 B2B 厂商（示例：某头部硬件厂商）的深圳制造业销售——每天补 10 家"大而近期有信号"的新制造客户（绝不撞已在库的两千多家），同时定期对手里存量客户做价值分层、把资源压到 S/A 级。**方法开源共享，你的客户名单本地保管、绝不入库。**

**落地模式 = 中心化生产 + 全员分发**（与「销售企微运营」一致）：1 人（销售运营/负责人）按节奏跑本 Skill 产出生鲜线索包/研判结论 → 进团队汇总表/云文档 → 全员跟进攻单。

---

## 两种模式（同一引擎，两种入口）

| | **拓客模式** | **研判模式** |
|--|--------------|--------------|
| 入口 | 信号驱动，发现新候选 | 指定公司（存量/目标/竞品客户） |
| 强排除 | **必做**（剔除在库存量） | 不做（对象本就在视野内） |
| 打分卡 | 可选（用于本批分层） | **必做**（核心产出） |
| 出口 | "新客 + 价值层级"线索包 | "经营价值结论 + 资源建议" |

> 无论哪种模式，都先产出 7 节调研报告（事实基底），再跑打分卡。区别只在入口与出口。

## 何时使用
- 拓客："做今日拓客""找 10 家新客户""筛选近期有融资/招聘扩张的公司""帮我做客户调研""撞库检查这家是不是已在库"
- 研判："评估这家客户经营价值""给存量客户做价值分层""这家值不值得重点跟""输出 S 级客户清单"
- 周期性自动化（如每个工作日 09:00 产出当日批次；或季度末做客户价值盘点）

## 核心工作流（6 步，两种模式共用底座）

### 第 1 步：定基准（参数化，由你提供）
- 目标区域（如"深圳"）、目标行业/赛道（可选）、规模门槛（如"上市/千人级/独角兽"）。
- 时间窗口：只抓"近 N 天"出现的新信号（招聘激增、新一轮融资、IPO 受理、重大中标/合同）。
- 在库名单路径：你团队的存量客户清单（公司名数组，JSON/Excel 均可）。**此文件含商业机密，绝不入库**（见 `.gitignore` 与 `references/强排除校验方法.md`）。
- （研判模式）指定要评估的公司清单。

### 第 2 步：候选采集（信号驱动，仅拓客模式）
围绕"有资金/有扩编 = 有采购"逻辑，从以下信号源捞候选（详见 `references/信号源清单.md`）：
- 招聘平台（BOSS/猎聘/前程无忧）近期大量招人的公司
- 融资/IPO 动态（一级市场融资、招股书受理、上市/递表）
- 招投标/重大合同/扩产新闻
初筛保留"规模达标 + 近期有时效信号"的候选。（研判模式跳此步，直接用指定公司）

### 第 3 步：强排除校验（核心，仅拓客模式必过）
对每一个候选，与你团队的**在库/存量客户名单**做强排除（算法见 `references/强排除校验方法.md` 与 `scripts/strong_exclusion.py`）：
1. **归一化**：去城市前缀（深圳市/深圳/广东）、去法律后缀（股份/有限公司）、去括号；
2. **双向子串匹配**：候选名 ⊇ 在库名 或 ⊂ 在库名 即命中；
3. **模糊兜底**：`difflib` 相似度 ≥ 0.85 视为命中；
4. **别名/母子公司穿透**：维护一张别名表，命中别名即判在库。
**零命中**才进入下一步；任何命中都直接剔除（避免"撞库/重复跟单"的硬红线）。

### 第 4 步：深度交叉验证（多源）
对候选/指定公司，做工商+招聘+融资多源核验，确认：
- **注册地/运营地**确实在目标区域（避免"惠州企业混进深圳"）；
- 规模、融资、招聘信号真实（企查查/天眼查 MCP 或 WebSearch 交叉验证）；
- 不与本批已选/历史已拓客重复（拓客模式）。

### 第 5 步：撰写 7 节调研报告（事实基底，两种模式共用）
按 `references/调研报告模板.md` 的 7 节结构写每家报告：
① 公司概况（含成立时间/人员）② 工商与股权（含工厂产能）③ 财务表现 ④ 招聘信号 ⑤ 融资/上市/股价信号（含未来趋势）⑥ IT 设备与算力需求 ⑦ 针对你公司产品线的切入建议。
- 公司名一律用**干净工商注册全称**（不带股票代码/品牌括号，注册名本身含地名括号的保留）；
- 每条结论附来源，宁缺毋滥。
> 7 节每个事实都对应打分卡一个维度（映射表见 `调研报告模板.md` 末尾），写好后即可直接打分。

### 第 6 步：跑打分卡 + 产出交付
1. **打分卡定级**（两种模式核心）：把 7 节事实转成 1–5 分，按 `references/商业价值评估维度与打分卡.md` 的权重算加权总分 → 判 S/A/B/C；可用 `scripts/value_score.py` 批量算（输入 `{公司:{维度:分}}` 的 JSON）。注意**红旗规则**：任一维度=1 分不得直接判 S，须人工复核。
2. **产出**：
   - 生成 `reports_YYYY-MM-DD.md`（源文件，全量 7 节）；
   - 转本地 `公司调研报告汇总_YYYY-MM-DD.docx`（见 `scripts/build_report.py`）；
   - 向团队汇总表（Excel，11 列含**价值层级**，结构见 `示例/拓客汇总表示例.md`）追加本批新行；用 `build_report.py --tier tier.csv` 把层级写入末列；
   - （研判模式）产出《价值研判报告》（7 节 + ⑧ 打分卡 + ⑨ 结论，模板见 `references/价值研判报告模板.md`）；
   - （可选）推送到云文档（腾讯文档/飞书/Notion）做分发；
   - 用 `present_files` 交付。

## 关键约束
- **强排除是硬红线（拓客模式）**：未零命中在库名单的，一律不得入选；误撞库曾导致"把已在跟的客户当新客报给领导"的严重事故。
- **公司名干净**：只用工商注册全称，不附加股票代码/品牌别名。
- **规模门槛**：小公司不纳入（"在精不在多"），每家公司做深度调研而非堆量。
- **脱敏**：本 Skill 不携带任何真实客户名单/真实公司数据；示例均为虚构，你的在库名单本地保管、绝不入库。
- **时效优先**：只选近期有新信号的，过期信号不入选。
- **打分卡统一标尺**：拓客与研判用同一张卡、同一套权重，保证"新客价值"和"存量价值"可比。

详见 `references/` 四份文档（强排除校验方法 / 调研报告模板 / 价值研判报告模板 / 商业价值评估维度与打分卡 / 信号源清单）与 `scripts/` 三个脚本（strong_exclusion / build_report / value_score）。
