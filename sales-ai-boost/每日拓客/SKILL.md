# 每日拓客引擎（sales-daily-prospecting）

## 这个 Skill 是什么
把销售"每天/每周找新客户"这件最耗精力的事，变成一套**可复用、可审计**的流水线：从海量信号里筛出"大且近期有采购迹象"的公司，严格排除你已经在跟的存量客户，产出一份能直接进团队客户池的《公司调研报告》+ 汇总表。

> 示例场景：某 B2B 厂商（示例：某头部硬件厂商）的深圳制造业销售，每天要补 10 家"大而近期有信号"的新制造客户，且绝不能撞上已在库的两千多家存量客户。本 Skill 把这套打法抽象成任何人可套用的流程——**流程与方法开源共享，你的客户名单本地保管、绝不入库**。

**落地模式 = 中心化生产 + 全员分发**（与「销售企微运营」一致）：1 人（销售运营/负责人）按节奏跑本 Skill 产出生鲜线索包 → 进团队汇总表/云文档 → 全员跟进攻单。

## 何时使用
- 用户说"做今日拓客""找 10 家新客户""筛选近期有融资/招聘扩张的公司""帮我做客户调研"
- 周期性自动化（如每个工作日 09:00 产出当日批次）
- 用户要补某行业/区域的客户池，或排查"这家是不是已经在库了（撞库检查）"

## 核心工作流（6 步）

### 第 1 步：定基准（参数化，由你提供）
- 目标区域（如"深圳"）、目标行业/赛道（可选）、规模门槛（如"上市/千人级/独角兽"）。
- 时间窗口：只抓"近 N 天"出现的新信号（招聘激增、新一轮融资、IPO 受理、重大中标/合同）。
- 在库名单路径：你团队的存量客户清单（公司名数组，JSON/Excel 均可）。**此文件含商业机密，绝不入库**（见 `.gitignore` 与 `references/强排除校验方法.md`）。

### 第 2 步：候选采集（信号驱动）
围绕"有资金/有扩编 = 有采购"逻辑，从以下信号源捞候选（详见 `references/信号源清单.md`）：
- 招聘平台（BOSS/猎聘/前程无忧）近期大量招人的公司
- 融资/IPO 动态（一级市场融资、招股书受理、上市/递表）
- 招投标/重大合同/扩产新闻
初筛保留"规模达标 + 近期有时效信号"的候选。

### 第 3 步：强排除校验（核心，必过）
对每一个候选，与你团队的**在库/存量客户名单**做强排除（算法见 `references/强排除校验方法.md` 与 `scripts/strong_exclusion.py`）：
1. **归一化**：去城市前缀（深圳市/深圳/广东）、去法律后缀（股份/有限公司）、去括号；
2. **双向子串匹配**：候选名 ⊇ 在库名 或 ⊂ 在库名 即命中；
3. **模糊兜底**：`difflib` 相似度 ≥ 0.85 视为命中（"示例迈步 vs 示例博迈"类字形误报需人工复核）；
4. **别名/母子公司穿透**：维护一张别名表（如"示例控股↔示例科技""示例装备↔示例车辆"），命中别名即判在库。
**零命中**才进入下一步；任何命中都直接剔除（这是避免"撞库/重复跟单"的硬红线）。

### 第 4 步：深度交叉验证（多源）
对通过强排除的候选，做工商+招聘+融资多源核验，确认：
- **注册地/运营地**确实在目标区域（避免"惠州企业混进深圳"类错误）；
- 规模、融资、招聘信号真实（企查查/天眼查 MCP 或 WebSearch 交叉验证）；
- 不与本批已选/历史已拓客重复。

### 第 5 步：撰写 7 节调研报告
按 `references/调研报告模板.md` 的 7 节结构写每家报告：
① 公司概况 ② 工商与股权 ③ 财务表现 ④ 招聘信号 ⑤ 融资/上市/股价信号 ⑥ IT 设备与算力需求 ⑦ 针对你公司产品线的切入建议。
- 公司名一律用**干净工商注册全称**（不带股票代码/品牌括号，注册名本身含地名括号的保留）；
- 每条结论附来源，宁缺毋滥。

### 第 6 步：产出与交付
1. 生成 `reports_YYYY-MM-DD.md`（源文件，全量 7 节）；
2. 转本地 `公司调研报告汇总_YYYY-MM-DD.docx`（便于归档/转发，见 `scripts/build_report.py`）；
3. 向团队汇总表（Excel，列结构见 `示例/拓客汇总表示例.md`）追加本批新行；
4. （可选）推送到你的云文档（腾讯文档/飞书/Notion）做分发；
5. 用 `present_files` 交付。

## 关键约束
- **强排除是硬红线**：未零命中在库名单的，一律不得入选；误撞库曾导致"把已在跟的客户当新客报给领导"的严重事故。
- **公司名干净**：只用工商注册全称，不附加股票代码/品牌别名。
- **规模门槛**：小公司不纳入（"在精不在多"），每家公司做深度调研而非堆量。
- **脱敏**：本 Skill 不携带任何真实客户名单/真实公司数据；示例均为虚构，你的在库名单本地保管、绝不入库。
- **时效优先**：只选近期有新信号的，过期信号不入选。

详见 `references/` 三份文档与 `scripts/` 两个脚本。
