# 强排除校验方法（Strong Exclusion Check）

> 目的：确保新筛出的候选公司，**绝不**出现在你团队的在库/存量客户名单里。这是"每日拓客"的硬红线——撞库（把已在跟的客户当新客）曾造成把存量客户报给领导、重复投入资源的严重事故。

## 一、为什么不能只做"精确匹配"

直觉做法：候选名字符串 == 在库名字符串 才判重复。这远远不够，真实名单里同一家公司有多种写法：

| 情形 | 例子（以下公司名均为虚构，仅演示匹配陷阱结构） | 精确匹配能判重吗 |
|------|------|----------------|
| 带括号/股票代码 | `示例光电(688001.SH)` vs 在库 `示例光电集团股份有限公司` | ❌ 判不出 |
| 品牌别名 | `DemoBot` vs 在库 `演示机器人科技集团股份有限公司` | ❌ 判不出 |
| 母子公司 | `示例车辆` vs 在库 `示例国际装备集团股份有限公司` | ❌ 判不出 |
| 城市前缀 | `深圳市示例科技` vs 在库 `示例科技` | ❌ 判不出 |
| 字形相近 | `示例迈步` vs 在库 `示例博迈`（不同公司） | ⚠️ 模糊匹配会误报，需人工复核 |

> ⚠️ 上表所有公司名均为**虚构占位**，仅用于演示五种匹配陷阱的结构，不代表任何真实公司、更不暗示"它们在某客户库里"。

所以必须做**归一化 + 双向子串 + 模糊兜底 + 别名穿透**四重校验。

## 二、四重校验算法

### 1. 归一化（Norm）
把候选名和在库名都清洗成"可比较的核心串"：
- 去掉括号及括号内内容：`(688001.SH)`、`（深圳）`、`（DemoBot）`
- 去掉城市前缀：`深圳市` / `深圳` / `广东` / `广东省` / `中国`
- 去掉法律后缀：`股份有限公司` / `有限公司` / `股份公司`（仅末尾）
- 去冗余空格

```python
def norm(s):
    s = re.sub(r'[\(（][^()（）]*[\)）]', '', str(s))
    for p in ['深圳市','深圳','广东','广东省','中国']:
        s = s.replace(p, '')
    s = re.sub(r'(股份)?有限公司$', '', s)
    s = re.sub(r'股份公司$', '', s)
    s = s.replace('（深圳）','').replace('(深圳)','')
    return re.sub(r'\s+', '', s).strip()
```

### 2. 双向子串匹配
归一化后，若 `候选核心串 ⊆ 在库核心串` 或 `在库核心串 ⊆ 候选核心串`，即命中。
> 这一步能覆盖"带股票代码/城市前缀/品牌括号"导致的长度差异——核心串相同时必被捕获。

### 3. 模糊兜底（difflib）
```python
ratio = difflib.SequenceMatcher(None, norm_cand, norm_inc).ratio()
if ratio >= 0.85:  # 命中
```
阈值建议 0.85。用于捕获缩写/错字/微调。

### 4. 别名/母子公司穿透
维护一张别名表（**本地保管，不入库**），命中别名即判在库：

```python
ALIAS = {
    '示例控股': ['示例科技集团', '示例科技', '示例华'],
    '示例装备': ['示例车辆', '示例天达', '深圳南方示例'],
    '示例长城': ['示例电脑', '示例'],
    # ……按需补充你公司的真实关系（本地保管，不入库）
}
```

## 三、误报处理（关键！）

`difflib ≥ 0.85` 会**误伤**：`示例迈步机器人` 与 `示例博迈机器人` 共享"机器人"等字，比例可能越线，但它们是**不同公司**。

**处理规范**：
1. 任何模糊命中，必须用"核心区别词"精确子串复核（如 `示例迈步` 是否在库、`示例博迈` 是否在库）。
2. 若区别词在库中**查无**，则为误报，放行。
3. 不确定时，**宁可剔除**候选，也不要冒撞库风险；或人工确认后再放行。

## 四、双源比对（提高召回）

在库名单可能有多个权威来源（如一份 Excel 客户池 + 一份系统导出 JSON）。**取并集**比对，避免某一来源漏记导致漏排。脚本 `scripts/strong_exclusion.py` 支持同时传入多个名单文件（`--list a.json --list b.xlsx`）。

## 五、机密说明（红线中的红线）

- 在库名单 = 商业机密，**绝不提交进任何公开仓库**。
- 本仓库的 `示例/incumbents_sample.json` 是**明显虚构**的演示数据，仅用于说明 JSON 结构（`{"names": ["...", ...]}`）。
- 真实名单放仓库外本地路径，运行时用 `--list` 传入。
- 根 `.gitignore` 已加规则（`在库客户名单*.xlsx`、`*客户名单*.json`、`incumbents*.json` 除 sample、`*拓客*.xlsx`），即使 `git add .` 也不会误提交真实名单。
- 别名表 `ALIAS` 同理含内部关系，本地维护、不入库。

## 六、脚本用法

```bash
# 准备候选（每行一个公司名）
cat > candidates.txt <<'EOF'
示例智能装备集团股份有限公司
演示科技股份有限公司
EOF

# 用演示假名单跑（默认）
python scripts/strong_exclusion.py --candidates candidates.txt

# 用你本地真实名单跑（名单在仓库外）
python scripts/strong_exclusion.py --candidates candidates.txt --list /你的/本地/在库名单.json

# 多来源并集比对
python scripts/strong_exclusion.py --candidates candidates.txt --list a.json --list b.xlsx
```

输出示例：
```
在库名单条数: 20 | 候选条数: 2
[通过] 示例智能装备集团股份有限公司
[命中] 演示科技股份有限公司
       在库: 演示科技集团有限公司
结果: 存在撞库，需剔除命中项
```
