# 联想企微周话题引擎（lenovo-wecom-weekly-topics）

## 这个 Skill 是什么
把销售"每周不知道发什么给客户"的难题，变成一条命令产出的**《本周话题包》**：5 个当周最新鲜、最相关的话题，每个都配好「可直接复制发企微」的话术卡片。

**落地模式 = 中心化生产 + 全员分发**：只需 1 人（运营/团队负责人）每周一跑一次本 Skill，产出话题包 → 发到企微群/腾讯文档 → 全体销售直接复制粘贴。销售端零部署、零学习。

## 何时使用
- 用户说"生成本周话题包""这周发什么给客户""来一批企微话题/话术"
- 每周一的定时自动化触发
- 用户要针对某类客户（如半导体/新能源）定制话题时

## 核心工作流（5 步）

### 第 1 步：确定本周基准信息
- 用 `date` 或 env 里的当前时间确定"本周"是哪一周、临近哪些节日/节气。
- 若用户指定了客户行业/分层，记下来做定向筛选。

### 第 2 步：联网抓取本周素材（WebSearch，全自动）
围绕 8 类话题矩阵，用 WebSearch 抓取**最近 7–14 天**的新鲜素材，每类至少搜 1 次：
1. **联想好消息**：`联想 最新 新品 财报 获奖 技术突破`（近1-2周）
2. **制造业行业趋势/政策**：`制造业 政策 利好 智能制造 数字化转型 2026`
3. **客户企业专属喜讯**：若用户给了目标客户名，搜其`中标/扩产/融资/上市`动态；否则跳过或做成"通用行业喜讯"
4. **政策补贴/以旧换新**：`设备更新 以旧换新 补贴 制造业 数字化 补贴政策`
5. **实用干货**：IT安全/降本/设备管理的当季 tips（可少量常青内容）
6. **节日/节气/关怀节点**：查当周临近的节日、节气、天气转折
7. **联想活动/权益**：`联想商用 活动 权益 优惠 企业购`（如无新活动可用常青权益）
8. **AI/算力热点**：`AI 大模型 算力 工作站 制造业 应用 2026`

抓取要点：优先官方/权威媒体来源；**记录来源链接**用于红线自检；宁缺毋滥，抓不到新鲜料的类别本周就不选。

### 第 3 步：筛选出本周 5 个话题
- 从 8 类里挑**当周最新鲜 + 与制造业客户最相关**的 5 个。
- **LRU 轮转（避免扎堆）**：读 `references/话题历史.md`，优先选最近 2–3 周未出现过的类别；8 选 5 时保证 8 类在滚动窗口内均匀覆盖，不连续两周同类别扎堆。
- 5 个话题尽量覆盖不同"承接商机"方向（产品切入 / 数字化算力 / 扩产采购 / 关系维系 / 转化钩子）。

### 第 4 步：按五段式话术公式生成话术卡片
每个话题生成一张卡片，严格套用 `references/话题话术库.md` 里的**五段式话术公式**和语气规则，字段固定为：
```
【话题N · 类别】标题
· 为什么现在发（时效点）：<一句话说明新鲜度/关联度>
· 适合客户：<行业/岗位>
· 话术（可直接发企微）：<X总，开场关怀 + 事件钩子 + 联想衔接 + 价值锚点 + 轻CTA，话术正文可带1–2 emoji>
· 承接动作：<客户回应后销售下一步做什么>
· 备用追问：<客户没回应时的第二句>
· 来源：<链接，仅内部留档，不发给客户>
```

### 第 5 步：红线自检 + 输出
逐条对照红线自检（见 `references/话题话术库.md`）：
- [ ] 不编造客户的具体事件（客户专属类必须有真实来源，否则改成行业通用话题）
- [ ] 不承诺具体价格/折扣数字（只说"有批量方案/优惠"）
- [ ] 每条话术只推一个核心信息
- [ ] 每条素材有可核实来源
- [ ] 关怀类（纯暖场）卡片只暖场、不硬推产品

输出：
1. 一份《本周话题包_YYYY-MM-DD.md》（按 `references/输出模板.md` 排版）到工作区。
2. 如用户要求，推送到腾讯文档「联想知识沉淀」便于分发；生成本地 docx 备份。
3. 把本周所选类别追加到 `references/话题历史.md`（供下周错开）。
4. 用 present_files 交付。

## 关键约束
- **口吻**：微信/企微口吻，亲切、精炼、重点突出，像"关心客户的销售朋友"，不是官方通稿。参考口语化、说人话的销售风格（少书面腔）。
- **时效性优先**：话题一定要是"本周最新"，过期新闻不要。
- **服务于商机**：每个话题最终都要能顺到"挖掘需求/促成沟通"，但纯关怀类只维系不推销。
- 详细的话题矩阵、话术公式、弹药库、种子范例见 `references/话题话术库.md`。
- 卡片排版格式见 `references/输出模板.md`。
